# しかくとり — マネタイズ＆ログイン設計

参考：Duolingo（フリーミアム＋サブスク／ゲスト開始）、日本の資格アプリ（スタディング=サブスク、過去問アプリ=試験ごと課金、宅建アプリ等）。

---

## 1. ログイン／アカウント設計（Duolingo流）

### 基本方針：「まず遊ばせて、あとで登録」
Duolingoは**起動直後にログインを求めない**。ゲストで学習開始→やる気が出た所で「進捗を保存しよう」と登録を促す。摩擦を減らし離脱を防ぐのが鉄則。

- **トップでログインは不要**（必須にしない）。現状どおりゲストで即開始、進捗は端末ローカル保存。
- **登録を促すタイミング**（Duolingoと同様、要所で自然に）：
  1. オンボーディング完了後に「アカウントを作って進捗を守る？（あとでもOK）」
  2. 1日の学習を終えた時・模試合格時（達成感が高い瞬間）
  3. 機種変更・複数端末で使いたい時
  4. **課金する時**（購入の復元・引き継ぎのため実質必要）
- 画面右上に常時「設定／ログイン」動線を置く（強制はしない）。

### ログイン手段（日本市場向け）
優先度順：
1. **Sign in with Apple**（iOS必須。他のソーシャルログインを出す場合、Appleの提供が審査で求められる）
2. **Google**
3. **LINE**（日本ユーザーに強い。友だち招待・通知とも相性良い）
4. **メールアドレス**（マジックリンク推奨＝パスワード不要で楽）

### 技術スタック（推奨）
- **Firebase Authentication**：Google / Apple / メールを標準サポート。無料枠大。
- **LINEログイン**：Firebaseの「汎用OIDCプロバイダ」または LINE Login SDK で連携。
- **進捗のクラウド同期**：**Cloud Firestore**（ユーザーごとに stats / wrong / goal を保存）。ゲスト時のローカル進捗は、登録時にクラウドへマージ。
- Flutterパッケージ：`firebase_auth` / `google_sign_in` / `sign_in_with_apple` / `cloud_firestore`。

## 2. マネタイズ設計

### モデル：フリーミアム（無料お試し）＋ 2種類の課金
コンテンツが「試験単位」なので、ユーザー案の**試験ごと買い切り**を主軸にしつつ、複数資格・回遊組み向けに**サブスク**を用意する二段構え。

#### (A) 無料でできること（お試し）
- 各試験、**各分野の最初の○問（合計30〜40問程度）**を無料開放。
- **模試を1回**だけ無料体験。
- 弱点復習・進捗機能は無料（アプリの価値を体感してもらう）。
→ 無料枠を解き終えると**ペイウォール**を表示。

#### (B) 課金プラン
| プラン | 種別 | 価格イメージ | 内容 |
|---|---|---|---|
| **試験ごと買い切り** | Non-Consumable | ¥600〜¥1,200 / 試験 | その試験の全問＋模試が無制限。目的資格が1つに決まっている人向け |
| **プレミアム（月額）** | Auto-renewable | ¥980 / 月 | 全62試験＋今後追加も全解放。広告なし |
| **プレミアム（年額）** | Auto-renewable | ¥5,800 / 年（実質 約¥480/月・40%お得） | 同上。年額で割安に誘導 |

- 価格心理：買い切りを数個買うより年額の方が得、と感じさせて回遊組をサブスクへ。
- **広告**：まずは**なし**でIAP一本を推奨（資格勉強アプリは広告過多が嫌われる）。将来、無料ユーザーの模試結果後などに控えめなインタースティシャルを検討。

#### 訴求（Duolingo流＋本アプリの強み）
ペイウォールで「**ムダなし勉強法で合格に近づこう。正解はもうやらない・まちがいだけ復習**」を前面に。無料枠終了時に「この試験、合格率○%。あなたの弱点だけ全問で潰そう」と背中を押す。

### 技術（IAP）
- **RevenueCat**を強く推奨（サブスク管理・レシート検証・復元・クロスプラットフォームが圧倒的に楽。無料枠あり）。Flutterは `purchases_flutter`。
- もしくは素の `in_app_purchase`（自前でレシート検証・権利管理が必要でやや大変）。
- Appleルール：デジタルコンテンツは**IAP必須**（外部決済リンク不可）、価格はApp Storeの価格帯、**購入の復元(Restore)必須**。
- 権利(entitlement)：買い切り=試験ID単位、サブスク=全解放フラグ。RevenueCatのentitlementで一元管理。

## 3. 実装フェーズ（おすすめ順）

- **Phase 0（現在）**：ゲスト運用・全問無料。iPhone実機＆Web公開済み。
- **Phase 1：課金の土台**
  - 無料枠ロジック（試験ごとに「無料N問まで」判定）＋ペイウォール画面。
  - RevenueCat導入、App Store Connectで商品登録（買い切り＋サブスク）。
  - Sandboxテスト。
- **Phase 2：認証＆同期**
  - Firebase Auth（Apple/Google/LINE/メール）＋Firestoreで進捗クラウド保存。
  - 「進捗を保存」導線を要所に。購入の端末間引き継ぎ。
- **Phase 3：継続の仕組み（Duolingo流）**
  - 連続学習ストリーク・学習リマインド通知・週間目標・ランキング等でリテンション向上（サブLTV↑）。
- **Phase 4：広告（任意）**・キャンペーン価格・初回割引など。

---

## 4. まず決めること
1. 無料枠の量（例：1試験あたり無料○問／模試○回）
2. 価格（買い切り単価・サブスク月/年）
3. ログイン手段の最終ラインアップ（Apple/Google/LINE/メール）
4. 事業者情報（App Store Connect法人/個人・プライバシーポリシー・特商法表記＝サブスクに必須）
