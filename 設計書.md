# 国家資格 学習クイズアプリ 設計書

> スマホで複数の国家資格を選び、過去問形式で出題 → 解答 → 正誤判定＋解説を表示する学習アプリ。

作成日: 2026-07-21 / バージョン: 0.2

## 決定事項（2026-07-21）
- **最初の資格：1級管工事施工管理技士**（→ プロファイルは §11）
- **問題データ：オリジナル作問**（過去問の論点を参考に、文言は自作。解説も自作）
- **アプリ形態：ネイティブアプリ**（App Store / Google Play 配信）

---

## 1. コンセプト

- **一言で**: 「いろんな国家資格の過去問を、スマホでサクッと解ける」
- **想定シーン**: 通勤中・スキマ時間に1問ずつ。resume（続きから）できる。
- **差別化ポイント**:
  - 複数資格を1アプリに横断（宅建・FP・簿記・危険物・登録販売者…など）
  - 弱点だけを自動で復習（間違えた問題の再出題）
  - 続けやすい（学習ストリーク・進捗の可視化）

---

## 2. コンテンツ方針（＝オリジナル作問に決定）

**問題・解説ともに自作**とするため、転載系の著作権リスクは基本的に回避できる。ただし作問時に守るべきルールがある。

### 作問ルール（重要）
- **過去問の文章をそのままコピーしない**。論点・出題テーマだけを参考に、文言・数値・事例は作り直す。
- 条文・法令・公式の数値など「事実」は引用可（著作物ではない）。条文番号や制度名は正確に。
- 解説も自作。市販問題集・予備校サイトの解説文をコピペしない。
- 各問題に `source` 欄で「参考にした論点・出典メモ・作成日」を残す（監査＆品質管理用）。

### 作問フロー（推奨）
1. 対象資格の「出題範囲・頻出論点」を洗い出す（公式の試験範囲＋過去の傾向）
2. 論点ごとにオリジナル問題を作成（AI下書き → 人が正確性を監修）
3. 正答・解説・難易度・分野タグを付与してJSON/DB化
4. 法改正チェック（法律系資格は改正で正答が変わる → 年1回見直し）

> オリジナル作問なので**質＝アプリの価値**。1資格を深く（例：まず100〜300問）作り込む方針。

## 3. スコープ（MVP → 拡張）

### MVP（まず動かす）
- **資格1つ**をオリジナル作問で収録（まず100問前後）
- 問題を出題（4択中心）→ 解答 → 正誤 → 解説表示
- 分野別で出題を絞れる
- 間違えた問題の記録＆再出題
- ログイン不要（進捗は端末に保存）＝オフラインで完結
- **ストア公開まで**（App Store / Google Play）

### フェーズ2
- ユーザー登録（進捗をクラウド同期／機種変で引き継ぎ）
- 学習ストリーク・正答率グラフ
- 「弱点集中モード」「本番シミュレーション（時間制限つき模試）」
- 資格の追加

### フェーズ3
- 有料プラン（広告なし／全資格解放）
- 記述・穴埋め問題など出題形式の拡張
- 学習リマインド通知

---

## 4. 主な機能一覧

1. **資格選択**：一覧から学習したい資格を選ぶ
2. **出題モード**
   - ランダム / 分野別 / 年度別 / 弱点のみ / 模試（時間制限）
3. **解答画面**：選択肢タップ → 即時 or まとめて判定
4. **正誤判定＋解説**：正解・不正解と根拠解説、関連論点
5. **復習キュー**：間違えた問題を自動で溜め、後日再出題（間隔反復）
6. **進捗ダッシュボード**：正答率、分野別の得意/苦手、連続学習日数
7. **しおり（続きから）**：中断地点を保存

## 5. 画面構成（スマホ前提・縦画面）

```
[起動]
  └─ ホーム（資格を選ぶ / 続きから / 進捗）
       ├─ 資格詳細（分野一覧・出題モード選択）
       │     └─ 出題設定（分野・問題数・モード）
       │           └─ 問題画面 ←→ 解説画面（次へ）
       │                 └─ 結果サマリー（正答率・復習に追加）
       ├─ 復習（間違えた問題キュー）
       ├─ 進捗・統計
       └─ 設定（ダーク/ライト・通知・アカウント）
```

### 問題画面のUX要点
- 1画面1問。指の届く下部に選択肢。
- 解答後に「解説」がスッと開く（アコーディオン）。
- 「わかった/自信ない」ボタンで復習頻度を調整（任意）。
- 進捗バー（3/10問）。

## 6. データモデル（論理設計）

```
Qualification（資格）
  id, name, slug, description, category, icon

Category（分野/科目）
  id, qualification_id, name, order

Question（問題）
  id, qualification_id, category_id,
  year(出題年度), question_type(single/multi/truefalse),
  body(問題文), choices[](選択肢), correct_answer,
  explanation(解説), difficulty, source(出典/規約メモ)

UserProfile（利用者）  ※フェーズ2でクラウド化
  id, display_name, plan(free/paid)

Attempt（解答履歴）
  id, user_id, question_id, selected_answer,
  is_correct, answered_at, response_ms

ReviewItem（復習キュー）
  id, user_id, question_id, due_date, ease(反復間隔)

StudySession（学習セッション）
  id, user_id, qualification_id, mode, started_at, score
```

- 問題データは **JSON or DB** で管理。MVPは資格ごとの JSON ファイルでも十分。
- `source` フィールドに「この問題の出典・利用可否」を必ず記録（後の権利トラブル防止）。

### 問題データ（JSON例）
```json
{
  "id": "kankouji1-houki-001",
  "qualification": "kankouji-1",
  "category": "法規",
  "subcategory": "建設業法",
  "type": "single",
  "is_applied": false,
  "body": "建設業法に関する次の記述のうち、誤っているものはどれか。",
  "choices": ["選択肢1", "選択肢2", "選択肢3", "選択肢4"],
  "correct": 3,
  "explanation": "選択肢3が誤り。建設業法○条により…（自作解説）",
  "difficulty": 3,
  "source": "オリジナル作問（参考論点：建設業法の許可 / 作成 2026-07-21）"
}
```
> `is_applied`: 「施工管理法（応用能力）」問題フラグ。模試モードの合格判定で使用。

## 7. 技術構成（＝ネイティブアプリに決定）

### フレームワーク：**Flutter** を推奨
- 1つのコードで iOS / Android 両対応。UIが作りやすく、クイズUIとの相性が良い。
- （代替案：React Native。Web資産を流用したいなら候補）

### 構成
- **アプリ本体**: Flutter（Dart）
- **問題データ**: MVPはアプリ同梱JSON（オフラインで全問解ける＝ネイティブの強み）
- **進捗保存**: 端末ローカル（`shared_preferences` / `sqflite`）→ ログイン不要で開始
- **フェーズ2でクラウド**: Supabase / Firebase（進捗同期・機種変引き継ぎ・ログイン）
- **課金**: `in_app_purchase`（App内課金）でサブスク/買い切り
- **通知**: ローカル通知で学習リマインド（プッシュはフェーズ2）

### ストア配信で必要になるもの（事前準備）
- Apple Developer Program（年 $99）／Google Play Console（初回 $25）
- アプリアイコン・スクリーンショット・プライバシーポリシー・審査対応
- **iOSのビルドには Mac が必要**（お手元のMacでOK）

> ネイティブなのでMVPでも「ストア公開」までが1マイルストーン。まず1資格を100問前後仕上げてから申請するのが現実的。

## 8. 出題・採点ロジック（コア）

- **出題選定**
  - ランダム：未出題を優先 → 全部解いたら復習キューへ
  - 弱点：正答率の低い分野・間違えた問題を重み付け抽出
  - 間隔反復（SRS）：`ReviewItem.due_date` が来た問題を出題
- **採点**
  - single/truefalse：選択と `correct` を照合
  - multi：全選択一致で正解
- **復習キュー更新（簡易SRS）**
  - 正解 → 次回出題を先送り（ease↑、due_dateを延ばす）
  - 不正解 → 近日中に再出題（ease↓）

## 11. 資格プロファイル：1級管工事施工管理技士

> 建設・設備分野の国家資格。実施団体：一般財団法人 全国建設研修センター。
> ※以下の問題数・基準は令和6年度時点。**申請前に必ず公式で最新を確認**（試験改定・法改正がある）。

### 試験の全体構成
| 検定 | 形式 | アプリ対応 |
|---|---|---|
| **第一次検定** | 四肢択一（マークシート） | ◎ クイズ形式にそのまま合致 → **MVPはここに集中** |
| 第二次検定 | 記述式（施工経験記述ほか） | △ 自動採点が難しい → フェーズ2以降で「模範解答＋自己採点」型 |

### 第一次検定の出題区分（＝アプリの「分野」タグ）
1. **機械工学等**（一般基礎／電気・建築の基礎／空調設備／衛生設備／機械設備 など）
2. **施工管理法**（施工計画・工程管理・品質管理・安全管理）
3. **施工管理法（応用能力）** ← 合格に別基準あり（下記）
4. **法規**（建設業法／労働基準法／労働安全衛生法／建築基準法／消防法／その他関連法規）

### 出題・合格の要点（令和6年度）
- 出題73問中 **60問を解答**（必須38問＋選択35問中22問）
- **合格基準：全体60%以上（=36点）**。加えて**「施工管理法（応用能力）」は50%以上**が別途必要。
- → アプリは①分野タグに上記4区分を持たせ、②模試モードで「必須/選択」の構造と2つの合格基準を再現できると本番に近い。

### 作問の進め方（この資格向け）
1. 上記4区分 → さらに細分野（例：法規＝建設業法／安衛法／…）に分解
2. 細分野ごとに頻出論点を洗い出し、オリジナル4択を作成
3. 「応用能力」問題は別フラグを立てて管理（合格判定ロジックで使う）
4. 法規は**法改正の影響大** → 年1回の見直しをルール化（§2の作問ルール参照）
5. まずMVPとして各区分あわせて **100問程度** を目標に着手
